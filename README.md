# 🍔 Restaurant Order Management System

Проєкт реалізований у межах Лабораторної роботи №9 з дисципліни «Інженерія програмного забезпечення». Це повноцінний веб-додаток для керування замовленнями та доставкою їжі, побудований за архітектурною моделлю **«Клієнт-Сервер»**.

## Тема та мета

**Тема:** Проєктування та реалізація програмного продукту з використанням архітектурної моделі «Клієнт-Сервер».
**Мета:** Вивчити специфіку клієнт-серверної архітектури, спроєктувати та розробити систему, де клієнти можуть переглядати меню та робити замовлення, а персонал - обробляти їх через адміністративну частину.

## Технологічний стек

### **Back-End**

- **Runtime:** Node.js
- **Framework:** Express.js
- **Database:** JSON-based File Storage (використано модуль `fs` для збереження даних у файлах `menu.json` та `orders.json`)
- **Middleware:** CORS, Body-parser

### **Front-End**

- **Framework:** Next.js (React)
- **Data Fetching:** TanStack Query (React Query) - для синхронізації стану замовлень з сервером
- **Styling:** TailwindCSS
- **Components:** shadcn/ui
- **HTTP Client:** Axios

## Архітектура та реалізація

### **Серверна частина (OOP Approach)**

Бекенд спроєктований з використанням класів для чіткого розподілу відповідальності:

- `JSONFileManager` - базовий клас для роботи з файловою системою.
- `MenuManager` - клас для керування асортиментом страв.
- `OrderManager` - клас для обробки замовлень (створення, оновлення статусу, видалення).

**Основні API Endpoints:**

- `GET /menu` - отримання списку всіх страв.
- `POST /orders` - створення нового замовлення.
- `PATCH /orders/:id` - зміна статусу замовлення (наприклад, "Pending" -> "Ready").
- `DELETE /orders/:id` - видалення замовлення.

### **Клієнтська частина**

Реалізовано дві основні ролі користувачів:

1. **Клієнт:**
   - Головна сторінка з меню страв.
   - Кошик для накопичення вибраних позицій.
   - Сторінка оформлення замовлення (Checkout) з валідацією даних.
2. **Адміністратор:**
   - Панель керування замовленнями.
   - Оновлення статусу замовлення в реальному часі за допомогою React Query.
   - Видалення замовлень зі списку.

## 🚀 Як запустити проєкт

1. **Клонуйте репозиторій:**

```bash
git clone https://github.com/TockePie/ipz_labs.git
```

2. **Запустіть Back-End:**

```bash
cd backend
npm install
node index.js
```

Сервер запрацює на `http://localhost:5000`.

3. **Запустіть Front-End:**

```bash
cd frontend
npm install
npm run dev
```

Додаток буде доступний на `http://localhost:3000`.

## Оновлення проєкту

У фінальній версії було видалено бібліотеку `React Hook Form` для спрощення обробки форм та переходу на використання вбудованих можливостей керованих компонентів React та стандартних HTML5 API для збору даних. Це дозволило зробити клієнтську частину легшою та продуктивнішою.
